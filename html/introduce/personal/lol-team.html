롤 내전 팀 구성 시스템 (WWWScrim)

롤 내전 팀 구성 시스템 바로가기
PERSONAL PROJECT

"사회자 없는 자동화 시스템:
노쇼 방지와 실시간 비밀 투표의 기술적 구현"

디스코드 봇과 수동 사회자에 의존하던 롤 내전 운영의 비효율을 기술로 자동화한 웹 서비스입니다. 실시간 API 연동과 비밀 투표 시스템으로 사회자 없이도 원활한 내전 진행이 가능하도록 설계했습니다.

역할 기획 · UI/UX · 프론트엔드
유형 실시간 웹 애플리케이션
기술 Next.js · React · JavaScript · Riot API

WWWScrim 메인 화면 — 팀 구성 및 매칭 인터페이스

01

기획 의도 — 디스코드 봇과 수동 진행의 한계를 넘어서

롤 내전이 돌아가는 방식을 경험하면서 발견한 가장 큰 비효율은 시스템의 '복잡함'과 '수동성'이었습니다. 기존에는 디스코드 봇을 이용해 복잡한 명령어를 입력해야 했고, 모든 과정을 관리하는 '사회자'가 반드시 필요했습니다.

특히 사회자가 일일이 개인 메시지를 받아 투표를 취합하거나, 인원이 다 왔는지 수동으로 확인하는 과정이 매우 번거로웠습니다. "사회자 없이도 웹에서 버튼 몇 번으로 모든 과정이 자동으로 돌아갈 수는 없을까?"라는 의구심에서 시작하여, 이를 기술적으로 해결하고자 했습니다.

문제 정의

기존 방식: 디스코드 봇 명령어 + 사회자 수동 관리 + 개인 DM 투표 취합
목표: 사회자 없이 웹 버튼 몇 번으로 전 과정 자동화

02

핵심 로직 1 — 노쇼 방지를 위한 실시간 API 체크

내전 운영에서 가장 골치 아픈 문제는 약속된 인원이 오지 않는 '노쇼'입니다. 이를 해결하기 위해 Riot API를 활용한 자동 체크 시스템을 구상했습니다.

  • Riot API 연동 자동 체크

    실제 롤 계정 정보와 연동하여 해당 유저가 게임에 접속했는지, 준비가 되었는지를 API로 실시간 판단

  • 자동 노쇼 판별 로직

    API를 통해 게임 시작 상태가 확인되지 않으면 '노쇼'로 간주하여 다음 단계로 넘어가지 못하게 차단

  • 데이터 기반 시스템으로 전환

    사회자가 육안으로 확인하던 불확실한 과정을 데이터 기반의 확실한 자동 시스템으로 치환

Riot API 연동 로직 구조 — 유저 접속 상태 실시간 확인 플로우

03

핵심 로직 2 — 완전 자동화된 실시간 게임 진행 시스템

기존 방식에서는 사회자가 유저들에게 일일이 개인 메시지를 받아 투표를 집계하고, 게임 진행 방식을 결정해야 했습니다. 이 과정을 완전히 자동화하여, 모든 유저가 방에 입장하는 순간부터 실시간으로 자동 진행되도록 설계했습니다.

게임 진행 방식은 세 가지 중 선택할 수 있습니다: 현재 팀들로 게임 진행, 방장이 투표로 게임 진행, 팀짜기를 투표로 게임 진행. 공정성을 위해 익명 투표로 진행되며, 방장도 한 명의 게임 유저로서 다른 유저들과 동일하게 투표에 참여합니다. 사회자의 개입 없이 시스템이 자동으로 결과를 처리하고 다음 단계로 진행합니다.

게임 진행 방식 선택 화면

04

제작 과정 — 알고리즘의 장벽을 AI로 돌파

실시간 데이터 연동과 복잡한 팀 매칭 알고리즘은 독자적으로 짜기에 기술적 난도가 매우 높았습니다. 알고리즘 전문가가 아니라는 사실을 인정하고, AI에게 구현하고자 하는 기능의 목적과 로직을 상세히 설명했습니다.

AI가 전체적인 알고리즘과 API 처리 코드를 짜주면, 그 코드를 분석하여 전체 시스템 흐름 내에서 의도대로 작동하는지 확인하고 적재적소에 배치했습니다. 스스로 모든 코드를 치지 못하더라도, AI를 파트너 삼아 복잡한 기술적 장벽을 돌파하고 시스템을 완성해 나가는 실무적 생산성을 경험한 프로젝트입니다.

AI 활용 방식

기능의 목적과 로직을 명확히 전달 → AI가 제시한 코드를 분석 → 시스템 흐름에 맞게 검증 및 배치

기술적 도전

실시간 API 연동, 팀 밸런싱 알고리즘, 비밀 투표 시스템 등 복잡한 기술적 요구사항을 AI 협업으로 해결

05

현재 상태 및 회고

현재 이 프로젝트는 기획했던 핵심 로직들이 구현되어 있으나, 여전히 수정하고 보완해야 할 점이 많다고 판단하고 있습니다. 특히 반응형 웹 환경을 더욱 견고하게 다듬어야 할 필요성을 느끼고 있으며, 실제 환경에서 발생할 수 있는 기술적 예외 사항들을 검토하며 고쳐나가는 과정에 있습니다.

이 프로젝트는 "생소했던 수동 시스템의 페인 포인트를 기술로 자동화해 보려 했던 도전" 그 자체로 큰 의미가 있습니다. 필요한 기술(API)을 어떻게 활용할지 고민하고 AI를 전략적으로 활용해 목표한 기능을 구현해 내는 실행력은 큰 자산입니다.

이 프로젝트에서 증명한 역량

생소한 도메인의 페인 포인트를 스스로 발견하고, 기술적 해결책을 설계하며, AI 협업을 통해 난이도 높은 기능을 실제로 구현해낸 문제 해결 능력과 실행력
